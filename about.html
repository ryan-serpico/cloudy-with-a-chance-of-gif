About Cloudy With a Chance of Gif

Why I made this

Back when I was in high school, besides being known as that guy who carries around Steve Jobs’ biography with him everywhere he goes, I was also known for my fascination with weather apps. Around that time, weather apps were a playground of sorts for app designers to toy around with how weather information was displayed.

Apps like Solar, Weather Line and my favorite, Dark Sky always caught my attention as I swiped through my Twitter timeline during class. I would see what people were saying about them, what neat little animations they had, and would eventually buy them. Yes, buy them. With real American dollars.

My classmates and teachers never understand my interest in them. The fact that I always had around 20 weather apps on phone at any given time baffled them, and they were confounded when I would eventually tell them that I spent actual money on many of them, especially when there was already a perfectly fine weather built into iOS.

What they didn’t get was that I couldn’t care less about the weather. When you live in south Florida, you can assume that its going to be hot as hell and be right 95% of the time. What really interested me was what people could do with this weather data. How could one present weather in a novel way? What animations could be incorporated to pique someones interest? Most importantly, how could they do all of this in a way that still conveyed data in an efficient matter?

Some of those same people that thought I was crazy for being so into weather apps eventually came around to seeing it my way. I even got some oohh’s and aahh’s when I showed them how the rain droplets in Solar would fall at an angle if you tilt your phone in a particular direction.

Cloudy With a Chance of Gif is my contribution back to the community of web and native app developers that delighted me with their work in the past. Hopefully one day, this web app is stumbled upon by some high school student in the world who is interested to see what can done with just some weather data.

What I used

  • First and foremost, I don’t think I would’ve committed to doing this project without first coming across Danial Shiffman’s Youtube channel. After watching his tutorials on Open Weather Map’s API and another on Giphy’s, plus one on incorporating user input, I felt pretty confident that I’d be able to take on this challenge.
  • Because Shiffman used it in his tutorials, I had to use p5.js, a javascript library meant for artists and those new to coding.
  • Bootstrap to make things look somewhat pretty.
  • jQuery, mainly for bootstrap.
  • Obviously Open Weather Map and Giphy’s API documentation pages.
  • Countless stackoverflow discussion threads.
  • This tutorial by Paula Borowska to style the search box to make it nicer.
  • JSON Formatter.
  • Font Awesome for their icons.
  • Documentation for the Open Graph protocol
  • w3’s documentation on how to make favicons.
  • Last, but certainly not least, professor McAdams, my intro to web apps professor.

What I Learned

  • This project gave me a crash course in how APIs work. Now, I’m no API expert at this point, but I’ll definitely be more comfortable in future projects that use them.
  • Even though I didn’t wind up using this piece of knowledge in the final project, I learned how to access array items inside of a json file.
  • This should’ve been obvious to me, but you can’t have two different functions with the same name. It clearly won’t work.
  • I learned a little bit about another javascript library, p5.js.
  • I learned never to use a javascript library you have absolutely no experience with.
  • I learned how to prevent mobile browsers from zooming into a form by using user-scalable=0. It’s something I see some sites I visit that frustrates the life out of me. It makes the web page ugly, and I didn’t want to have any of that on something I worked so hard on.
  • I learned how to have the form submit the query by hitting the enter key on the keyboard rather than using a button.
  • Chrome is better for web development than Safari is. Sigh.
  • Speaking of Chrome, I used its network tools to monitor the API calls that were taking place. This eventually hinted me toward learning that an issue plaguing the web app for weeks was the fact that the Giphy API was being called right when the page loaded.
  • A basic understanding of what callback functions are and how they work.

Design decisions

Two design choices I feel like I need to explain is the lack of a submit button and the inclusion of the actual weather forecast rather than just the gif.

When it comes to the button, it didn’t fit into the search box I created and ultimately viewed it as being unnecessary. At the top of the main page when a user arrives, it tells the user to hit enter. It may just be me, but I never move my cursor just to click a submit button ever, so I may be biased.

And for the inclusion of the actual weather description, I found it important to do so for the fact that the gif that is returned sometimes doesn’t relay the actual weather outside. Sometimes it’s totally irrelevant, other times its misleading. When I was showing this off to one of my friends, a gif that was returned for clear says returned a gif of the Simpsons title with snow floating by it.

What I want to do in the future

  • Have it setup where the user doesn’t have to refresh the page and enter in a city again to see another gif or to see the weather for a different city. This probably involves AJAX, which I don’t have time to learn right now.
  • Search suggestions. I already have come across a stackoverflow thread on it, but I don’t have time to work on it in this initial release.
  • Polishing the design more.
